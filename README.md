# x

用自然语言写命令行命令。

```
m3 :: ~/.config ‹main*› » x create an empty file named test
m3 :: ~/.config ‹main*› » touch test  # generated by x
```

在 zsh 里以 `x ` 开头输入自然语言并回车，`x` 会把生成的命令放回命令行（不会直接执行），
确认无误后再按一次回车执行。原始的自然语言会保留在 history 里。

## 安装

### Nix

```sh
nix profile install github:yangtau/x
# 或在 flake 中引用 inputs.x.url = "github:yangtau/x";
```

### Cargo

```sh
cargo install --path .
```

## 配置

1. 登录（使用 ChatGPT 订阅，OAuth，无需 API key）：

   ```sh
   x auth login
   ```

   会打印一个登录 URL，在浏览器中完成授权即可（回调监听本机 1455 端口，
   与 Codex CLI / openclaw 相同的方式）。

2. 在 `~/.zshrc` 末尾加入：

   ```sh
   eval "$(x init zsh)"
   ```

### 可选配置

`~/.config/x/config.toml`：

```toml
provider = "openai-chatgpt"   # 目前唯一支持的 provider
model = "gpt-5.2-codex"
reasoning_effort = "low"       # minimal | low | medium | high
```

zsh 集成的环境变量（在 `eval` 之前设置）：

| 变量 | 默认 | 说明 |
|---|---|---|
| `X_TRIGGER` | `x` | 触发前缀，可改成 `@` 等任意前缀 |
| `X_TAG` | `# generated by x` | 追加在生成命令后的注释，置空禁用 |

也可以不装 shell 集成直接用：`x gen -- "find large files"` 或 `x find large files`。

## 设计说明

### 触发方式：为什么是前缀词 + 拦截 accept-line

shell 集成通过重载 zsh 的 `accept-line` widget 实现：回车时检查原始 `$BUFFER`，
若以触发前缀开头，则把前缀之后的**原始文本**交给 `x gen`，用生成的命令替换
buffer。因为拿到的是 zle 的原始 buffer，自然语言从未经过 shell 解析，所以
`?`、`*`、引号、空格都是安全的。

调研过的备选方案：

- **`!` 前缀**：与 zsh 的历史展开（history expansion）语义冲突，且很多插件
  / 用户肌肉记忆依赖 `!!`、`!$`，不采用。
- **`#` 前缀**：zsh 交互模式默认不允许注释（需 `interactive_comments`），且
  语法高亮插件会把整行当注释涂灰，体验差，不采用。
- **`@` 前缀**：可行（buffer 拦截不经过解析），可通过 `X_TRIGGER=@` 启用。
- **无前缀纯自然语言**：需要对每一行输入做"是命令还是自然语言"的分类，误判
  会拦截正常命令且每次回车都有延迟，不采用。
- **快捷键方案**（[shell_gpt](https://github.com/ther1d/shell_gpt) 的做法，
  Ctrl+L 替换当前 buffer）：同样基于原始 buffer，可行，但前缀词更符合
  "像敲命令一样敲自然语言"的直觉，故默认采用前缀词。

注意：与其他同样重载 `accept-line` 的插件（少见）组合时，加载顺序靠后者生效。

### Provider 抽象

`src/provider/mod.rs` 定义 `Provider` trait（输入自然语言 + shell/os/cwd 上下文，
输出单行命令），由 `config.toml` 的 `provider` 字段选择实现。目前唯一实现
`openai-chatgpt`：

- 认证走 OpenAI 官方 Codex 公共客户端的 OAuth PKCE 流程
  （`auth.openai.com`，本机 1455 回调），即 openclaw / opencode 接 ChatGPT
  订阅的同一套机制，token 存于 `~/.local/share/x/auth.json`（0600），
  过期前自动 refresh。
- 请求打到 `chatgpt.com/backend-api/codex/responses`（Responses API、
  SSE-only、`store: false`，需 `ChatGPT-Account-Id` header），按订阅计费，
  不消耗 API 余额。

新增 provider（如 Anthropic、本地模型）只需实现 trait 并在
`provider::from_config` 注册。

### Shell 拓展

`x init <shell>` 输出对应 shell 的集成脚本，目前仅 zsh
（`src/shell/x.zsh`）；新增 shell 在 `src/shell/` 加脚本并在
`init_script` 注册即可。
